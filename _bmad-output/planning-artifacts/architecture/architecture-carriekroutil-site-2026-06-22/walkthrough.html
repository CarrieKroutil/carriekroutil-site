How the site is built

A fast, static personal site whose entire architecture exists to protect one habit: write markdown, git push, it's live. Everything below is in service of that — and of it still being true in three years.

The paradigm

Build-time static generation, content-as-source. There is no server and no runtime state. The git repository is the single source of truth; every deploy is reproducible from source. Content is markdown, the presentation layer is project overrides shadowing the Hextra theme, and delivery is one Amplify pipeline. Authoring touches only content/ — everything downstream is mechanical.

The 13 rules

These are the invariants — the calls a future post or layout file couldn't safely guess. ADOPTED means reality or an upstream doc already settled it.

AD-1 The authoring loop is inviolable adopted

Rule Publishing is exactly write markdown → git push. No CMS, login, web editor, or manual copy step ever enters the path. Any feature that adds a manual authoring step is a regression.

AD-2 Exactly one deploy pipeline adopted

Rule AWS Amplify is the only pipeline; it builds from source on push to the default branch only and serves at carriekroutil.com via Route 53. Branch/PR previews off; local hugo server is the only pre-publish preview. One environment: production.

AD-3 Canonical post front-matter contract

Why The highest-divergence-risk surface on the site — if two posts name fields differently, the build quietly breaks.

Rule Every post carries YAML: title, date (bad/missing fails the build), tags (lowercase), draft, optional featured, and an optional hero: {src, alt} map — the one named hero field, also read by the OG tags. Read-time is derived ("{n} min read", floor 1), never typed.

AD-4 Post stream is a type: blog section at /posts/ adopted

Rule Posts live in content/posts/ with _index.md setting type: blog — keeps the /posts/{slug}/ URLs and activates Hextra's blog layouts. Slug = bundle folder name; front-matter slug: only as a deliberate override; the title never sets the URL.

AD-5 Customization lives in project overrides, never in the module

Rule Custom partials, layouts, and CSS live in the project's own layouts/ and assets/, shadowing the Hextra module. The module is read-only and never edited in place — so theme upgrades don't clobber your work.

AD-6 One card renderer for home, list, and tags

Rule A single custom partial (post-card.html) renders the card design ported from the UX home.html mockup, and is the one renderer used by the home grid, the /posts/ list, and tag pages — so they can never drift into three designs. Hextra doesn't auto-populate a latest-posts feed; this partial does the query.

AD-7 Featured post is a front-matter flag, single slot

Rule A post sets featured: true. Home renders exactly one slot: candidates sorted by date descending, first taken (deterministic if more than one); falls back to most-recent when none flagged. Pinning stays markdown-only.

AD-8 Topic filtering is tag term-pages, no client JS

Rule A topic chip links to /tags/{topic}/. Hugo core generates and filters the term page; Hextra ships no styled term layout, so a custom _default/term.html styles it as a post list (reusing the AD-6 card). No in-place JS filtering of the stream.

AD-9 Page bundles; alt enforced; processing deferred

Rule Each post is a page bundle (images beside the markdown). Alt text is enforced, not requested — a render-image.html markup hook fails the build on empty alt. Lazy-loading is native. Resize/WebP processing is deferred (the markup hook still ships in v1).

AD-10 Versions pinned; upgrades deliberate

Rule Hugo extended pinned via HUGO_VERSION; Hextra locked in go.mod. Upgrades happen on purpose (hugo mod get -u + re-test), never automatically — so a content-only push can't be broken by a release you didn't choose.

AD-11 amplify.yml is the single source of build truth

Rule One file at the repo root: preBuild fetches the module (hugo mod get), build runs hugo --gc --minify at the pinned version, artifacts = public/.

AD-12 Build failure must notify, never silently no-op

Rule Amplify's native email-on-failure is on. Amplify only swaps on success, so a failed build keeps the prior good site live and emails you. A clean build with no new post is distinguishable from a failure.

AD-13 Core reading works without JavaScript adopted

Rule Home, posts, stream, about, and tag pages render as static HTML with JS off. FlexSearch is the single accepted JS exception and degrades gracefully. System fonts only — no web-font network load.

Deferred — on purpose

None are launch gates. Naming them is half the contract.

Analytics tool
GoatCounter vs Cloudflare Web Analytics — a single privacy-friendly include, chosen at/just after launch. Revisit at launch; don't let it drift.
Light-mode AA
Re-verify the three accents on Hextra's light surfaces before relying on light mode. Dark is the AA-guaranteed v1 path.
Image resize/WebP
A custom render hook using Hugo image processing — added only if images get heavy.
Custom 404
On-brand "wandered off the fairway" copy. Non-blocking.
Legacy deploy cleanup
Disabling the old S3-sync action + GCP artifacts in ~/code/quickstart — a manual one-time task, outside this repo.
