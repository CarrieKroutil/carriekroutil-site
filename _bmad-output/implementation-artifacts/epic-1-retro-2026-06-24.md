## What went well

- **One settled pipeline, zero remnants.** FR-11 was met cleanly — no second deploy path in-repo, no GitHub Action, no stale brand artifacts. The "boring, reliable" NFR-4 goal held.
- **Pinned-versions discipline** (Hugo via `HUGO_VERSION`, Hextra in `go.mod`) set the longevity tone for the whole project up front.
- **Docs-as-deliverable for infra stories.** 1.5/1.6/1.7 each had irreducible manual AWS-console steps only Carrie can perform; treating the documented checklist in SETUP.md as the deliverable was the right call and was reused for the rest of the project.

## What was hard / what we deferred

- **Light-mode WCAG AA is unverified and deliberately deferred.** The fuchsia accent measures ~3.4–3.9:1 on Hextra's white surface, below the 4.5:1 AA floor. Decision: dark is the AA-guaranteed v1 path; light ships untuned. This is the single biggest carried-forward item and it spawned two dependent cosmetic deferrals (double focus ring on Hextra-managed controls; phantom border on the blur layer) that only matter once light mode is tuned.
- **Sequencing call:** 1.6/1.7 were deferred to after Epic 2 because they verify against the post stream/page Epic 2 builds, and publish-by-push was already proven by the live site. This worked — no rework — and became the model for batching.

## Lessons

1. **Infra stories are documentation stories.** When the only remaining work is a console toggle in Carrie's AWS account, the checklist *is* the deliverable. Don't force code where none exists.
2. **Name deferrals explicitly and link them.** The light-mode AA chain stayed coherent across four stories because each deferral cited the others. Keep that habit.
3. **Pin early.** Locking Hugo/Hextra versions in Story 1.1 paid off all the way through launch.
