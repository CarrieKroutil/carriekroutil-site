{{- /* custom/post-tags.html — THE clickable topic-chip renderer. Reused by the single post (Story 2.2), the post-card (Story 2.3 — all of the post's tags), and the term page (Epic 4). Do not duplicate chip markup elsewhere. The optional `limit` (below) is kept for callers that want a capped set; the card passes the bare Page, so it shows all tags. Each chip links the real taxonomy term page (/tags/{topic}/) via .GetTerms, so links stay correct once Epic 4 styles the term page. Labels are lowercase as authored; CSS uppercases. Accent tint is assigned by the topic's position in the site-wide alphabetical tag list, cycling the three brand accents — so a topic keeps the SAME color everywhere it appears (single page, card, term page), adjacent chips never match, and all three colors show up even with only a few topics. (A plain hash-mod-3 clustered 3 of 4 topics onto one color.) Context (one of): - a Page -> all of the page's `tags` terms - dict { page: , limit: N } -> the page's terms, capped at N (0/absent = all) - dict { terms: } -> render exactly these taxonomy term Pages (used by the Home hero, Story 3.1, to chip the site's topics straight from the taxonomy — same renderer, same per-topic tint, so colors match everywhere). */ -}} {{- $terms := slice -}} {{- $limit := 0 -}} {{- if reflect.IsMap . -}} {{- if isset . "terms" -}} {{- $terms = .terms -}} {{- else -}} {{- $limit = .limit | default 0 -}} {{- $terms = .page.GetTerms "tags" -}} {{- end -}} {{- else -}} {{- $terms = .GetTerms "tags" -}} {{- end -}} {{- if and $limit (gt (len $terms) $limit) -}}{{- $terms = first $limit $terms -}}{{- end -}} {{- $tints := slice "violet" "fuchsia" "amber" -}} {{- /* Site-wide alphabetical tag list — gives every topic a fixed global position so its tint is stable across pages, while cycling the accents avoids the hash-collision clustering. */ -}} {{- $names := slice -}} {{- range $name, $_ := site.Taxonomies.tags -}}{{- $names = $names | append $name -}}{{- end -}} {{- $names = sort $names -}} {{- with $terms -}}
