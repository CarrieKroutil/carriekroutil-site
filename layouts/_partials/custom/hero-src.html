{{- /* custom/hero-src.html — single source of truth for resolving a page's hero into a render-ready, responsive image. Returns a dict { url, srcset, width, height }. WHY THIS EXISTS: four templates (post-card, project-card, posts/single, projects/single) each resolved the hero the same way by hand. That duplication lives here now, and this is the one place image processing happens — tune sizing/quality once and every surface follows. RESPONSIVE, NOT FIXED-SIZE (this is the important part): a single fixed-width variant looks soft the moment a card renders wider than it — on 2-up desktop and especially on HiDPI/retina where a 450px card is 900+ physical pixels. So local RASTER heroes emit a `srcset` of several WebP widths and the browser downloads only the one that fits its actual box and pixel density. Small bytes AND crisp at every size. Callers supply a `sizes` attribute describing the layout. - widths are capped to the source's real width, so we never upscale (which is what caused the blur). If the source is smaller than every requested rung, we still emit one variant at native width so it's never under-served. - quality defaults high (q90) because these heroes are smooth gradient art, where lower WebP quality shows visible banding. The variants are tiny regardless (a 1200px gradient is ~8-12KB), so the quality is cheap. Override per call if needed. - width/height are the LARGEST variant's dimensions (all variants share one aspect), so a caller can set them to reserve space and avoid layout shift (CLS). Pass-through (returns url only; srcset empty, width/height = 0): - SVG heroes — vector, scales infinitely and Hugo's raster pipeline can't Resize them. Served as-is (crisp at any size). Share images swap in a raster sibling via social-image.html. - External http(s) heroes — not local resources, nothing to process. - Declared-but-missing local files — url stays empty so the caller renders no . Input dict: { page, widths (slice of ints, ascending), quality (int, optional) } */ -}} {{- $page := .page -}} {{- $widths := .widths -}} {{- $q := .quality | default 90 -}} {{- $url := "" -}} {{- $srcset := "" -}} {{- $w := 0 -}} {{- $h := 0 -}} {{- with $page.Params.hero -}} {{- $r := $page.Resources.GetMatch .src -}} {{- if $r -}} {{- $ext := path.Ext .src | lower -}} {{- if in (slice ".png" ".jpg" ".jpeg" ".webp" ".tif" ".tiff" ".bmp") $ext -}} {{- $maxW := $r.Width -}} {{- /* Build one WebP per requested width that the source can honor without upscaling. */ -}} {{- $variants := slice -}} {{- $keptTop := 0 -}} {{- range $widths -}} {{- if le . $maxW -}} {{- $variants = $variants | append ($r.Resize (printf "%dx webp q%d" . $q)) -}} {{- $keptTop = . -}} {{- end -}} {{- end -}} {{- /* Source smaller than our top rung → also offer native width so it's not under-served (never ABOVE the top rung — that ceiling is deliberate, for bytes). */ -}} {{- $topRung := index (last 1 $widths) 0 -}} {{- if and (lt $keptTop $maxW) (lt $maxW $topRung) -}} {{- $variants = $variants | append ($r.Resize (printf "%dx webp q%d" $maxW $q)) -}} {{- end -}} {{- /* Nothing kept (source narrower than the smallest rung) → single native-width variant. */ -}} {{- if not $variants -}} {{- $variants = slice ($r.Resize (printf "%dx webp q%d" $maxW $q)) -}} {{- end -}} {{- $set := slice -}} {{- range $variants -}}{{- $set = $set | append (printf "%s %dw" .RelPermalink .Width) -}}{{- end -}} {{- $largest := index $variants (sub (len $variants) 1) -}} {{- $url = $largest.RelPermalink -}} {{- $srcset = delimit $set ", " -}} {{- $w = $largest.Width -}} {{- $h = $largest.Height -}} {{- else -}} {{- /* SVG (or any non-raster) local resource: serve as-is, unprocessed. */ -}} {{- $url = $r.RelPermalink -}} {{- end -}} {{- else if hasPrefix .src "http" -}} {{- $url = .src -}} {{- end -}} {{- end -}} {{- return (dict "url" $url "srcset" $srcset "width" $w "height" $h) -}}
